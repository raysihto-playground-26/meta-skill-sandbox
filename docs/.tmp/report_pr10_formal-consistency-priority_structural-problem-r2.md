# PR #10 を契機に顕在化した「形式整合性優先」の構造問題レポート（R2 改訂版）

## 本書の目的

本書は、PR #10 に付随して観測されたレビューコメント（`discussion_r2841492177`）を「きっかけ」として、より構造的な問題――**AI directive file が「形式的整合性（Formal Consistency）」を最優先してしまい、結果として「推論効率（Inference Efficiency）」を損ねる**――を、課題認識として共有するためのレポートである。

本 R2 版では、R1 版で示した「S/N比・トークン効率」の議論をさらに根拠付ける、**本質的な大前提**を明示的に追加した。

---

## 本質的な大前提（R2 追加）

### 設計思想の核心

この `SKILL.md` の設計において、何より先に認識すべき大前提は次である。

> **「AI に"確実に伝えるべき最重要指示"だけを、最小コスト（最小コンテキスト）で、安定して効かせる設計へ寄せる」**

この一文が示すように、目標は「完全性（誤読ゼロ・網羅）」ではない。**非決定性を前提にした実務的な落とし所**として、

- **コンテキスト肥大を抑えつつ**、それによって**重要指示の"すっぽかし"を最小化**し
- **「ここだけは絶対」を漏らさない**

という **"効きやすさの最適化"** を最優先する。

### 構造化された設計思想

| 階層 | 内容 |
| ---- | ---- |
| **最上位ゴール** | 指示が AI に正しく（＝意図どおりに）伝わり、実行されること |
| **前提認識** | LLM は非決定的であるため、完全性を目標にしても費用対効果が悪い |
| **実務的ゴール① 指示密度の向上** | 「重要度の高い指示」を最小限の組で持たせる（指示密度を上げる） |
| **実務的ゴール② S/N 向上** | 冗長な説明や自明な補足で注意資源を散らさない（S/N を上げる） |
| **バランス条件** | 短くしすぎて重要指示が抜けるのも NG、形式整合性・網羅のために膨らませて注意を逸らすのも NG |
| **最適化の狙い** | 「最小の強制力」＋「最小の文量」で「失敗確率が下がる領域」を狙う |

### この大前提が示すこと

- **完全性は目標ではない**。LLM が非決定的である以上、「あらゆるケースで誤読ゼロ」を達成しようとすると、コンテキストが膨大になり、重要指示が埋没する逆効果を招く。
- **効きやすさが唯一の評価軸**。ある記述が「形式上の完全性を高めるか」ではなく、「AI が重要指示を正確に実行する確率を上げるか」で評価される。
- **Attention 最適化としての指示圧縮**。AI agent の注意資源（Attention）は有限である。自明な定義や形式的なメタ規則にその資源を費やさせることは、重要指示への Attention を奪う行為に等しい。
- **非決定性前提の実務最適**。「どうすれば確実に伝わるか」ではなく、「どうすれば失敗確率が最も下がるか」という視点が正しい問いである。

## 結論

- 本件は、単一の形式不備（例: `always` の定義欠落）を修正するという話ではない。
- 根本は、AI directive file の構造が **形式的整合性を最優先に置く設計**になっており、その結果として
  - **トークン効率が劣化**し
  - **S/N比（信号対雑音比）が低下**し
  - **AI agent の推論が「ルールの意図」ではなく「形式の整合性」へ誘導される**
    という、目的に反する挙動を誘発する点にある。
- したがって、本件の致命度は高い。なぜなら、このツール（`SKILL.md`）の存在意義――**AI agent が適切な指示を正しく解釈・実行する**――を損ない、**効果が得られない／逆効果**となる可能性が高いからである。

---

## 前提（この `SKILL.md` に期待する最優先事項）

この `SKILL.md` は、AI agent が指示を解釈・実行するためのツールとして用意されている。したがって評価軸の最優先は次である。

- **AI が重要指示を実行する確率（効きやすさ）の最大化**
- **そのための手段としての、トークン数の節約**
- **そのための手段としての、S/N比（信号対雑音比）の向上**

ここでいう S/N比は、AI agent の入力コンテキストにおいて「重要なルール解釈に寄与する信号」と「寄与しない（むしろ邪魔をする）雑音」を区別し、**推論に効く情報密度を最大化する**ことを指す。

「トークン節約」と「S/N向上」は、あくまで「効きやすさの最大化」を達成するための手段である。目的と手段を混同してはならない。

## 観測された事象（きっかけとしての `discussion_r2841492177`）

対象レビューコメント: `https://github.com/raysihto-playground-26/meta-skill-sandbox/pull/10#discussion_r2841492177`

レビューコメントは、次の主旨を指摘している（要約）。

- `condition_identifiers` の解釈が「`conditions` の識別子は `definitions` と厳密一致（MUST）」を要求している
- 一方で `conditions: ["always"]` が多数利用されるのに `definitions` に `always` がない
- したがって仕様が自己矛盾（self-violation）している

このコメント自体が正しいか否かより重要なのは、**この種の指摘が"妥当"として成立してしまう構造**である。

補足（パスの実体）:

- 実体は `.agents/skills` にあり、他の `.*/skills`（例: `.cursor/skills`）はそこを指すシンボリックリンクである。

## 深刻性の根拠：なぜ「形式整合性優先」は危険か

本件を深刻と判断する理由は、「形式的不整合を指摘されること」そのものではない。次のような **構造的な副作用**が、目的（効きやすさの最大化）を直接毀損するためである。

### 1) 自明語・予約語レベルまで definitions を強制すると、効きやすさが必ず落ちる

`conditions` の識別子に対し「`definitions` との厳密一致（MUST）」を課す構造は、自然に次を誘発する。

- `always` のような自明な共通語まで、**definitions への明示を強制**しがちになる
- その結果、definitions が **辞書化**し、入力が冗長化する

これは「ルール本体（行動に直結する指示）」の占有比率を下げ、コンテキストウィンドウ内での **信号密度を下げる**。

**本質的大前提との接続**: `always` を `definitions` に追加することは、形式完全性を満たすが、`always` は自明語であり定義に情報価値がない。追加トークンは重要指示への Attention を奪い、失敗確率を上げる方向に働く。これは設計の最上位ゴールに直接反する。

### 2) 注意（Attention）の分散：重要度の低い辞書的定義に注意資源が割かれる

AI agent の推論で本当に重要なのは、「いまこのタスクで何を守るべきか」という優先順位の抽出である。

ところが、definitions に自明語が大量に蓄積すると、モデルは **解釈上の重要度が低い要素**にも注意資源を割く可能性が高くなり、相対的に「守るべきルール本体」の解釈精度が落ち得る。

**本質的大前提との接続**: Attention 最適化としての指示圧縮という観点では、自明語の定義追加は純粋なノイズ（負の信号）である。

### 3) 長文化により重要情報が埋もれる（いわゆる "Lost in the Middle" を誘発し得る）

入力が長くなるほど、相対的に重要な情報が埋もれやすくなる。形式整合性を満たすための定義・メタ規則が増えるほど、肝心の「タスク遂行に必要な制約」が中間に埋没しやすくなる。

（参考文献）長文コンテキストにおける重要情報の取りこぼし傾向として "Lost in the Middle" が知られている:

- Liu et al., "Lost in the Middle: How Language Models Use Long Contexts" (2023)

**本質的大前提との接続**: 完全性追求によるコンテキスト肥大は、「重要指示のすっぽかし」確率を直接上げる。これは非決定性前提の実務最適という観点からみて最悪のトレードオフである。

### 4) 形式警察化：AI agent の出力が「内容」ではなく「形式整合性」の最適化に寄る

AI directive file が「形式上の完全性」を強く要求し続けると、AI agent は

- 形式上の MUST 違反の回避（definitions の穴埋め等）
- 用語や識別子の厳格化
- スキーマ整形

といった **形式作業**を優先しやすくなる。これは、ツールの目的である「適切な指示の解釈・実行」という **推論の本丸**から注意を逸らす。

**本質的大前提との接続**: 「最小の強制力＋最小の文量で失敗確率が下がる領域を狙う」という方針に対して、形式警察化は真逆の方向に作用する。

## この問題が「存在意義をなくす」理由

本来、`SKILL.md` は AI agent にとって「推論を助ける圧縮された規範（高S/Nの指示）」であるべきである。

しかし、形式整合性を最優先する構造が強いと、次の逆転が起こりうる。

- **本末転倒**: 推論のためのツールが、形式整合性のためのツールになってしまう
- **逆効果**: ルールの遵守精度を上げるはずの文書が、冗長化とノイズで遵守精度を下げる
- **コスト増・遅延増**: 不要トークンの増加は、そのまま推論コストと応答遅延の増加に直結する

よって、本件は「バグ」ではなく、**設計目標の取り違え（最適化対象の誤り）**に近い問題であり、致命度が高い。

## 合意すべき課題認識

本書で共有したい課題認識は次である。

- **本質的大前提（追加）**: 目標は形式完全性ではなく「最小コンテキストで重要指示を安定的に効かせること」であり、LLM の非決定性を前提にした設計最適化が必要である
- **形式的整合性（Formal Consistency）を"最優先"に置く構造は不適切である**
- **効きやすさ（重要指示の伝達安定性）と S/N比を"最優先"に置くことを、構造として明示しなければならない**
- `discussion_r2841492177` は、その構造問題が表出した一例（きっかけ）にすぎない
